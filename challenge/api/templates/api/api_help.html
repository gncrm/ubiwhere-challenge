Occurrence API Help

API for managing occurrences in a urban environment. The responses are in the JSON format.
All the requests are made with the information the API requires sent as parameters.
The available URLs and methods are:

api/

Displays the help menu of this API

api/user-registration/

POST - Registers a new user in the API. Parameters:

  • username: The desired username
  • admin_pw (OPTIONAL): Code necessary to register admin accounts

api/occurrences/

GET - Returns the list of ocurrences (no parameters) or a filtered list of occurrences if the request has search parameters:

  • latitude: Latitude
  • longitude: Longitude
  • radius: Maximum distance to the point sent in the request
  • username: Username of the user that posted the occurrences
  • category: Category of the occurrences to be shown. Possible categories: CONSTRUCTION, SPECIAL_EVENT, INCIDENT, WEATHER_CONDITION, ROAD_CONDITION
POST - Adds a new occurrence. Parameters:
  • description: Description of the occurrence
  • latitude: Latitude of the occurrence
  • longitude: Longitude of the occurrence
  • user_token: User API token
  • category: Category of the occurrence. Possible categories: CONSTRUCTION, SPECIAL_EVENT, INCIDENT, WEATHER_CONDITION, ROAD_CONDITION

api/occurrences/{id}/

GET - Returns the ocurrences with the ID in the URL (no parameters)
PUT - Updates the status of the occurrence with the ID in the URL (ADMIN only). Parameters:

  • user_token: User API token (must be an admin to use this functionalty)
